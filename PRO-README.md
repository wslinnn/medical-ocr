## 📖 使用指南

### 首次使用

1. **配置 API Token**
   - 点击右上角设置或使用默认Token输入框
   - 输入您的 AI Studio API Token
   - 点击"保存"按钮

2. **上传病历文件**
   - 方式1: 点击"选择文件夹"批量上传
   - 方式2: 点击"选择文件"单个上传
   - 方式3: 直接拖拽文件/文件夹到上传区域

3. **开始识别**
   - 文件添加到队列后，点击"开始全部识别"
   - 等待识别完成

4. **查看和编辑结果**
   - 在"记录"视图查看所有识别结果
   - 点击"对比"查看原图和结果
   - 点击单元格直接编辑内容
   - 标记记录状态（待审核/已审核/需复核）

5. **导出数据**
   - 选择要导出的记录
   - 点击"导出选中"或"导出全部"
   - 选择保存位置

## 📊 数据说明

### 识别结果字段

- **姓名**: 患者姓名
- **性别**: 男/女/--
- **年龄**: 数字年龄
- **诊断**: 识别的诊断内容
- **分期**: 疾病分期
- **状态**: pending/reviewed/flagged/corrected
- **识别时间**: 自动记录的时间戳

### 状态说明

- **待审核 (pending)**: 默认状态，新识别的记录
- **已审核 (reviewed)**: 已人工确认无误
- **需复核 (flagged)**: 标记有问题需要复核
- **已修正 (corrected)**: 已修正错误的记录

## 🐛 故障排除

### 问题1: 应用无法启动

**解决方案**:
1. 检查 Node.js 版本（建议 v16+）
2. 删除 `node_modules` 重新安装
3. 检查 `main-pro.js` 路径是否正确

### 问题2: 识别失败

**解决方案**:
1. 检查 Token 是否正确
2. 检查网络连接
3. 确认 API 服务是否可用
4. 查看控制台错误信息

### 问题3: 图标不显示

**解决方案**:
1. 在 `assets/` 目录放置 `icon.png` 和 `tray-icon.png`
2. 参考 `assets/README.md` 准备图标

### 问题4: 数据丢失

**解决方案**:
1. LocalStorage 有容量限制（约5-10MB）
2. 建议定期使用"导出全部数据"功能备份
3. 未来版本将支持 SQLite 数据库
