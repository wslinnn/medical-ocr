## 🔴 P0 - 严重 Bug (必须修复)

### BUG-001: 事件监听器内存泄漏
**模块**: `pro-compare-render.js:156-208`
**严重程度**: P0 - 严重
**类型**: 内存泄漏

**问题描述**:
每次调用 `updateCompareView()` 时，都会为图片容器添加新的 `wheel` 事件监听器，但不会移除旧的监听器。长时间使用会导致内存泄漏。

**复现步骤**:
1. 上传多个文件
2. 在对比视图和记录视图之间多次切换
3. 查看浏览器任务管理器，内存持续增长

**问题代码**:
```javascript
// pro-compare-render.js:196-207
const imgContainer = document.getElementById('image-container');
if (imgContainer) {
    imgContainer.addEventListener('wheel', (e) => {  // 每次都添加新监听器
        if (e.ctrlKey) {
            e.preventDefault();
            const delta = e.deltaY > 0 ? -10 : 10;
            state.imageZoom = Math.max(25, Math.min(300, state.imageZoom + delta));
            updateCompareView();
        }
    });
}
```

**修复建议**:
在添加新监听器前，先移除旧监听器，或在初始化时只添加一次。

### BUG-002: record.id 类型精度丢失
**模块**: `pro-events-batch.js:11`, `pro-table.js:49`
**严重程度**: P0 - 严重
**类型**: 数据精度问题

**问题描述**:
record.id 使用 `Date.now() + Math.random()` 创建，产生浮点数。使用 `parseFloat()` 转换时可能丢失精度，导致无法正确匹配记录。

**问题代码**:
```javascript
// 创建id
id: Date.now() + Math.random()  // 1712345678901.1234

// 使用时
parseFloat(cb.dataset.id)  // 可能变成 1712345678901.123
```

**影响范围**:
- 批量操作无法正确选中记录
- 删除操作可能删除错误的记录

**修复建议**:
使用 `Math.floor(Date.now() + Math.random() * 1000000)` 或改用字符串ID。

---

### BUG-003: 分页边界条件未处理
**模块**: `pro-events-pagination.js:18-23`
**严重程度**: P0 - 严重
**类型**: 边界条件错误

**问题描述**:
当 `totalPages` 为 0 时，分页逻辑会失效。用户无法进行任何分页操作。

**问题代码**:
```javascript
const totalPages = state.pagination.totalPages;
if (state.pagination.currentPage < totalPages) {
    state.pagination.currentPage++;
}
```

**修复建议**:
添加 `totalPages > 0` 的检查。

## 🟡 P1 - 中等 Bug (应当修复)

### BUG-004: 删除记录后索引越界
**模块**: `pro-events-compare-filter.js:33-41`
**严重程度**: P1 - 中等
**类型**: 数组越界

**问题描述**:
当用户在对比视图删除当前记录后，`state.compareIndex` 可能超出 `filteredRecords` 的范围。

**复现步骤**:
1. 进入对比视图
2. 筛选显示3条记录
3. 删除第2条记录
4. `compareIndex` 仍为1，但filteredRecords只有2条

**修复建议**:
删除操作后检查并调整 `compareIndex`。

### BUG-005: contenteditable 无输入验证
**模块**: `pro-compare-render.js:111-128`
**严重程度**: P1 - 中等
**类型**: 数据验证缺失

**问题描述**:
可编辑字段没有长度限制和字符过滤，用户可以输入：
- 超长文本破坏布局
- HTML/JavaScript 代码（XSS风险）
- 特殊字符导致解析错误

**修复建议**:
添加 `maxlength` 属性和输入验证。

### BUG-007: 去重逻辑过于简单
**模块**: `pro-data-dedup.js:22-28`
**严重程度**: P1 - 中等
**类型**: 业务逻辑缺陷

**问题描述**:
只按 `姓名 + 诊断` 去重，没有考虑：
- 同一天同一患者的多次检查
- 相同姓名不同患者的记录

**问题代码**:
```javascript
const key = `${record.name}|${record.diagnosis}`;
```

**修复建议**:
使用 `姓名 + 诊断 + 日期(YYYY-MM-DD)` 作为去重键。
