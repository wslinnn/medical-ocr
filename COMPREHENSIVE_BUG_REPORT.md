## 🚨 P0级别Bug (严重 - 必须立即修复)

### 代码逻辑 - P0

#### BUG-L-001: localStorage解析异常未处理
**文件**: `js/pro-state.js:11`
**严重程度**: P0

**问题描述**:
使用 `JSON.parse()` 解析localStorage数据时，如果数据被破坏或格式错误，会导致整个应用崩溃无法启动。

**问题代码**:
```javascript
const state = {
    records: JSON.parse(localStorage.getItem('records') || '[]'),
};
```

**复现步骤**:
1. 打开浏览器开发者工具
2. 修改localStorage中的records为非法JSON: `{broken data`
3. 刷新页面
4. 应用完全无法启动

**修复建议**:
```javascript
try {
    const stored = localStorage.getItem('records');
    records = stored ? JSON.parse(stored) : [];
} catch (e) {
    console.error('数据损坏，使用空数组', e);
    records = [];
}
```

#### BUG-L-002: 并发修改导致状态不一致
**文件**: `js/pro-ocr.js:15-96`
**严重程度**: P0

**问题描述**:
处理队列时 `state.fileQueue` 和 `state.records` 被并发修改，没有使用锁机制，导致数据不一致。

**复现步骤**:
1. 添加多个文件到队列
2. 点击"开始识别"
3. 在处理过程中点击"批量删除"删除已识别的记录
4. 数据错乱或程序崩溃

**修复建议**:
- 使用队列快照进行迭代
- 添加全局锁机制防止并发操作
- 处理期间禁用修改state的UI操作

---

#### BUG-L-003: record.id冲突风险
**文件**: `js/pro-ocr.js:62`, `js/pro-queue-ops.js:27`
**严重程度**: P0

**问题描述**:
使用 `Date.now() + Math.random()` 生成ID，高并发时可能产生重复ID。

**修复建议**:
使用UUID或递增计数器保证唯一性。

### 交互 - P0

#### BUG-I-001: Delete键删除记录过于危险
**文件**: `js/pro-events-keyboard.js:94-111`
**严重程度**: P0

**问题描述**:
在对比视图按Delete键可直接删除记录，虽然有确认对话框，但对医疗数据来说太危险。

**复现步骤**:
1. 在对比视图查看记录
2. 按Delete键
3. 确认对话框后记录永久删除

**修复建议**:
- 改为"移到回收站"机制
- 或需要Shift+Delete才能直接删除
- 添加恢复功能

### 用户体验 - P0

#### BUG-UX-001: 缺少处理进度实时反馈
**严重程度**: P0

**问题描述**:
OCR处理过程中用户只能看到整体进度条，不知道当前处理哪个文件、还需多久。

**修复建议**:
- 显示当前处理的文件名
- 添加预计剩余时间
- 显示处理速度

---

#### BUG-UX-002: OCR失败信息不够详细
**严重程度**: P0

**问题描述**:
文件识别失败时只显示简单错误消息，用户无法了解具体原因和解决方案。

**修复建议**:
- 区分网络错误、文件格式错误、API错误
- 提供具体解决方案
- 添加"查看详情"功能

---

#### BUG-UX-003: 键盘导航支持不完整
**严重程度**: P0

**问题描述**:
- 没有可见的焦点状态
- Tab键导航顺序不明确
- 模态框焦点管理不当

**修复建议**:
- 为所有交互元素添加明显的焦点样式
- 优化Tab键导航顺序
- 模态框打开时焦点移至模态框内

---

#### BUG-UX-004: 清空数据二次确认不够明显
**严重程度**: P0

**问题描述**:
清空所有数据使用原生confirm对话框，样式不统一，容易被忽略。

**修复建议**:
使用自定义模态框，强调操作的不可逆性。
